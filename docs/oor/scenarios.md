# Сценарии работы (UI/UX Flow)

Описанное рабочее пространство OOR-IDE обеспечивает трассируемость: каждое действие в интерфейсе привязано к [Mandate](domains/oor_manager/Mandates.md) и при необходимости к [Transition](domains/oor_manager/Transitions.md).

---

## 1. Общая структура рабочего пространства

Рабочее пространство состоит из трёх основных зон:

```
+------------------+------------------------------------------+----------------------+
|  Навигация       |  Основная область                        |  AI-Sync             |
|  (проекты,       |  (редактор сущностей или визуализатор    |  (панель контекста   |
|   домены)        |   State Machine)                         |   для Cursor)        |
+------------------+------------------------------------------+----------------------+
```

- **Левая панель:** список проектов и доменных модулей; выбор текущего артефакта.
- **Центр:** либо экран редактора сущностей, либо визуализатор диаграммы состояний (переключение вкладкой или режимом).
- **Правая панель:** AI-Sync - сформированный контекст для копирования в IDE (Cursor).

Роль пользователя (Analyst / Reviewer / AI-Developer) определяется при входе; кнопки и поля отображаются/блокируются по мандатам.

---

## 2. Экран редактора сущностей

**Цель:** создание и редактирование проектов, доменных модулей, определений методов; выполнение переходов состояний (Draft - Validated - Frozen).

### 2.1. Список и выбор артефакта

- В левой панели отображаются **Requirement_Project** и вложенные **Domain_Module**.
- Выбор проекта открывает сводку (название, список модулей); выбор модуля открывает его в центральной области.
- Для каждого модуля отображается **текущий статус** (Draft / Validated / Frozen) и даты.

**Трассировка:** отображение списка - мандат M-OOR-READ-DRAFT или M-OOR-READ-VALIDATED в зависимости от статуса артефакта.

### 2.2. Форма редактирования (Domain_Module)

В центральной области:

- Поля: название, описание, ссылка на проект, статус (только для чтения при Validated/Frozen).
- Список связанных **Method_Definition** с возможностью добавления/удаления (если статус Draft и есть M-OOR-EDIT-DRAFT).
- Блок **действий по жизненному циклу** - кнопки в зависимости от текущего статуса и роли:

| Текущий статус | Кнопка | Мандат | Переход |
|----------------|--------|--------|---------|
| Draft | "Утвердить" | M-OOR-ACT-APPROVE (Reviewer) | transition_approve |
| Validated | "Вернуть на доработку" | M-OOR-ACT-REJECT | transition_reject |
| Validated | "Зафиксировать версию" | M-OOR-ACT-FREEZE (Reviewer) | transition_freeze |

Кнопки отображаются только при наличии у текущего пользователя соответствующего мандата; при отсутствии прав - скрыты или неактивны.

### 2.3. Редактирование Method_Definition и проектов

- Отдельные формы для **Method_Definition** (имя, описание, rule_schema) и для **Requirement_Project** (название, описание, список модулей).
- Правила те же: создание и изменение только при M-OOR-CREATE / M-OOR-EDIT-DRAFT и статусе Draft у родительского артефакта.

**Критерий приёмки сценария:** каждая кнопка в интерфейсе прослеживается до конкретного Mandate и при необходимости до Transition (см. ТЗ и [Mandates](domains/oor_manager/Mandates.md)).

---

## 3. Визуализатор State Machine (Mermaid)

**Цель:** наглядное отображение жизненного цикла (Draft - Validated - Frozen) и допустимых переходов.

### 3.1. Источник данных

- Текст диаграммы берётся из [Transitions](domains/oor_manager/Transitions.md) или из сгенерированного по нему описания (тот же Mermaid-код, что в спецификации).
- В прототипе допустимо загружать статический фрагмент из `domains/oor_manager/Transitions.md` или из API, возвращающего этот фрагмент.

### 3.2. Отображение

- В центральной области доступна вкладка **"Диаграмма состояний"**.
- Рендер Mermaid (библиотека [mermaid.js](https://mermaid.js.org/) или аналог): отображение состояний Draft, Validated, Frozen и переходов approve, reject, freeze.

### 3.3. Трассировка

- Под диаграммой или в тултипе: при клике на переход показывать код мандата и роль (например: "Переход approve - мандат M-OOR-ACT-APPROVE, роль Reviewer"). Это связывает визуализатор с [Mandates](domains/oor_manager/Mandates.md) и [Transitions](domains/oor_manager/Transitions.md).

---

## 4. Панель AI-Sync (контекст для Cursor)

**Цель:** передача актуального контекста OOR-IDE в ИИ-агент, чтобы ИИ понимал структуру требований и домена.

### 4.1. Содержимое панели

Панель формирует текст, включающий по выбору пользователя:

- **Глоссарий:** ключевые термины из [glossary](glossary/README.md) (Domain, Mandate, Traceability Chain, Invariants, [Base] Document).
- **Текущий артефакт:** выбранный проект или доменный модуль (название, статус, связанные сущности) в структурированном виде.
- **Жизненный цикл:** краткое описание переходов Draft - Validated - Frozen (из Transitions) и кто имеет право выполнять переходы (из Mandates).
- **Структура проекта:** дерево `docs/oor/` (index, roles, glossary, domains/oor_manager, scenarios) для навигации ИИ по репозиторию.

### 4.2. Действия пользователя

- **Копировать в буфер:** одна кнопка - сформированный контекст копируется в clipboard; пользователь вставляет его в чат Cursor.

### 4.3. Роль и мандат

- Доступ к панели и копированию контекста имеют роли Analyst, Reviewer, AI-Developer (чтение утверждённого контекста - M-OOR-READ-VALIDATED; для черновиков - M-OOR-READ-DRAFT у Analyst/Reviewer). В MVP достаточно считать, что любой авторизованный пользователь может использовать AI-Sync для текущего выбранного артефакта.

**Критерий AI-Ready:** настройка `.cursorrules` дополняет этот сценарий: ИИ в IDE сможет постоянно опираться на структуру `docs/oor/` и терминологию OOR-IDE без ручного копирования при каждом сеансе.

---

## 5. Краткая сводка сценариев

| Сценарий | Основные элементы | Трассировка |
|----------|-------------------|-------------|
| Редактор сущностей | Список проектов/модулей, формы редактирования, кнопки переходов | Кнопки - Mandate + Transition |
| Визуализатор State Machine | Mermaid-диаграмма из Transitions | Переход на диаграмме - Mandate, роль |
| Панель AI-Sync | Глоссарий, текущий артефакт, переходы, структура docs/oor | Контекст для ИИ; .cursorrules |

Эти сценарии достаточны для реализации MVP рабочего пространства OOR-IDE

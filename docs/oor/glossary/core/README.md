# OOR Core Glossary

Назначение: Определение базовых абстракций системы OOR-IDE. Все термины здесь являются родительскими для доменных глоссариев. Повторное описание унаследованных свойств в доменных глоссариях запрещено.

## [Base] Document (Документ)

**Definition:** Зафиксированная информация, имеющая реквизиты для идентификации. Обладает номером, заголовком, автором и жизненным циклом.

**Business Attributes:**
- `RegistrationNumber` (UUID) - Уникальный номер документа, используется как первичный ключ
- `Title` (string) - Заголовок документа, краткое описательное название
- `Initiator` (UUID) - Идентификатор участника, создавшего документ
- `State` (enum) - Текущее состояние жизненного цикла: `Draft` | `Validated` | `Frozen`
- `CreatedAt` (ISO 8601 datetime) - Дата и время создания документа
- `UpdatedAt` (ISO 8601 datetime) - Дата и время последнего обновления документа

**Lifecycle States:**
- `Draft` (В разработке) - документ доступен для редактирования, может быть изменён создателем
- `Validated` (Утверждён) - документ проверен и утверждён Reviewer, доступен только для чтения
- `Frozen` (Зафиксирован) - документ зафиксирован как неизменяемая версия для релиза или архива

**Inheritance Note:** Является родителем для всех файлов спецификаций в OOR-IDE (Entities.md, Transitions.md, Rules.md, Mandates.md, roles/*.md, scenarios.md) и всех доменных сущностей (Requirement_Project, Domain_Module, Method_Definition, Authorization_Mandate).

## [Base] Participant (Участник)

**Definition:** Сущность, обладающая уникальной идентичностью, способная совершать действия и нести ответственность внутри системы.

**Business Attributes:**
- `IdentityToken` (UUID) - Уникальный идентификатор участника в системе
- `ContactPoint` (string) - Способ связи: email, телефон или другой идентификатор
- `DisplayName` (string) - Отображаемое имя участника в интерфейсе
- `Role` (enum) - Назначенная роль: `Analyst` | `AI-Developer` | `Reviewer`
- `IsActive` (boolean) - Признак активности участника в системе

**Inheritance Note:** Является родителем для всех ролей системы (Analyst, AI-Developer, Reviewer). Каждый участник может быть назначен на одну или несколько ролей.

## Domain (Домен)

**Definition:** Логически выделенная область системы с собственными сущностями, переходами, правилами и мандатами.

**Attributes:**
- `Name` (string) - Уникальное имя домена (например, `oor_manager`)
- `Description` (string) - Описание назначения домена
- `Entities` (Entity[]) - Список сущностей домена
- `Transitions` (Transition[]) - Список переходов между состояниями
- `Rules` (Rule[]) - Список правил (инвариантов) домена
- `Mandates` (Mandate[]) - Список мандатов доступа

**Example:** `oor_manager` - домен управления объектно-ориентированными требованиями.

## Mandate (Мандат)

**Definition:** Формализованное разрешение, которое домен выдаёт роли; единственный легальный способ доступа к данным или действиям внутри домена.

**Типология:**
- **Action Mandates (Мандаты на действие):** Дают право на инициацию конкретного метода (перехода состояния). Префикс ID: `M-[DOM]-ACT-`
- **Visibility Mandates (Мандаты на видимость):** Определяют "горизонт видимости" - какие атрибуты сущности доступны для чтения или редактирования. Префикс ID: `M-[DOM]-VIEW-`

**Attributes:**
- `Code` (string) - Уникальный код мандата (например, `M-OOR-ACT-APPROVE`)
- `Type` (enum) - Тип мандата: `Action` | `Visibility`
- `Target` (UUID) - Целевой объект (Transition для Action, Entity для Visibility)
- `Scope` (JSON) - Область действия (фильтры, условия)
- `Description` (string) - Описание назначения мандата

## Traceability Chain (Цепочка трассируемости)

**Definition:** Последовательность ссылок от элемента интерфейса до реализации: UI элемент → Мандат → Переход → Метод.

**Элементы цепочки:** Requirement - Domain/Entity - Transition - Mandate - Role - UI action / generated code.

Цепочка прослеживаемости обеспечивает:
- По любому требованию или мандату можно установить, **откуда** оно возникло и **где** используется (код, экран, переход).
- По кнопке или действию в интерфейсе OOR-IDE можно установить, к какому **Mandate** и **Transition** оно относится.

**Implementation:** Реализуется через ссылки в коде (комментарии, аннотации) и структуру данных OOR-IDE.

## Invariants (Инварианты)

**Definition:** Правила, которые должны выполняться всегда, независимо от состояния системы.

**Attributes:**
- `Code` (string) - Уникальный код правила (например, `R-MANDATE-TRANSITION-EXISTS`)
- `Description` (string) - Описание правила на естественном языке
- `Condition` (string/JSON) - Формальное условие проверки
- `Scope` (string) - Область применения правила (глобальная, доменная, сущностная)

**Примеры инвариантов (подробнее - в `domains/oor_manager/Rules.md`):**
- Мандат не может ссылаться на несуществующий переход (Transition).
- Переход возможен только из допустимого предыдущего состояния (например, Validated только из Draft).
- Сущность в статусе Frozen не может быть изменена без снятия статуса по правилам.

Инварианты проверяются при изменении данных и при выполнении переходов; их нарушение блокирует операцию или переход.

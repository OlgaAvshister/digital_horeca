# Глоссарий OOR-IDE

Семантический слой: единая терминология для всех артефактов системы.

## Наследование от [Base] Document

**Правило:** Все файлы спецификаций в OOR-IDE считаются наследниками типа **[Base] Document**.

Каждый такой артефакт должен:

- иметь **идентификатор** (например, UUID или стабильный путь в репозитории);
- относиться к **версии** или моменту актуальности (для прослеживаемости);
- быть пригодным для **трассировки** к другим артефактам (роли, переходы, мандаты).

Структура каталогов (`roles/`, `glossary/`, `domains/oor_manager/`, `scenarios.md`) и формат файлов (YAML, Markdown) являются реализацией этого правила: любой файл в этих разделах считается документом-спецификацией, наследуемым от [Base] Document.
